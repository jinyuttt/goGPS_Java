## 2. 单点定位与 RTK 定位

### 2.1 单点定位（伪距）

```java
import org.gogpsproject.GoGPS;
import org.gogpsproject.GoGPS.RunMode;
import org.gogpsproject.consumer.TxtProducer;
import org.gogpsproject.producer.parser.rinex.RinexObservationParser;
import org.gogpsproject.producer.parser.rinex.RinexNavigationParser;

// 1. 创建数据源
RinexObservationParser obs = new RinexObservationParser(
    new File("rover.obs"), new Boolean[]{true, false, false, false, false});
RinexNavigationParser nav = new RinexNavigationParser(new File("rover.nav"));
obs.init();
nav.init();

// 2. 创建输出
TxtProducer txt = new TxtProducer("./output/standalone.txt");

// 3. 创建 GoGPS 引擎（单点定位不需要基准站）
GoGPS goGPS = new GoGPS(nav, obs)
    .addPositionConsumerListeners(txt)
    .setCutoff(15)                              // 高度角截止角 15°
    .setDynamicModel(GoGPS.DynamicModel.STATIC);

// 4. 执行单点定位
goGPS.run(RunMode.CODE_STANDALONE);
goGPS.runUntilFinished();

// 5. 释放资源
obs.release(true, 10000);
nav.release(true, 10000);
```

### 2.2 RTK 定位（载波相位双差卡尔曼滤波）

```java
import org.gogpsproject.GoGPS;
import org.gogpsproject.GoGPS.RunMode;
import org.gogpsproject.consumer.TxtProducer;
import org.gogpsproject.producer.parser.rtcm3.RTCM3FileReader;
import java.io.File;

// 1. 创建数据源
String roverPath = "path/to/rover.rtcm3";
String basePath  = "path/to/base.rtcm3";
int gpsWeek = 2422;

RTCM3FileReader roverReader = new RTCM3FileReader(new File(roverPath), gpsWeek);
RTCM3FileReader baseReader  = new RTCM3FileReader(new File(basePath), gpsWeek);
roverReader.init();
baseReader.init();

// 2. 创建输出
TxtProducer txt = new TxtProducer("./output/rtk_result.txt");

// 3. 创建 GoGPS 引擎（三参构造：导航源, 流动观测, 基准观测）
//    注意：roverReader 同时提供导航星历和观测值
GoGPS goGPS = new GoGPS(baseReader, roverReader, baseReader)
    .addPositionConsumerListeners(txt)
    .setDynamicModel(GoGPS.DynamicModel.STATIC)
    .setCutoff(15)                              // 卫星高度角掩蔽角
    .setDebug(false);

// 4. 可选：高级配置
goGPS.setIonoOpt(GoGPS.IonoOpt.IFLC);           // 双频消电离层组合
goGPS.setTropOpt(GoGPS.TropOpt.EST);            // 实时估计天顶对流层延迟
goGPS.setAmbiguityRatioThreshold(3.0);           // LAMBDA Ratio 检验阈值
goGPS.setCycleSlipThreshold(0.05);               // 周跳检测阈值 (m)

// 5. 执行 RTK 解算
goGPS.run(RunMode.KALMAN_FILTER_CODE_PHASE_DOUBLE_DIFF);
goGPS.runUntilFinished();

// 6. 释放资源
roverReader.release(true, 10000);
baseReader.release(true, 10000);
```

**GoGPS 核心配置项：**

| 方法 | 说明 | 默认值 |
|---|---|---|
| setCutoff(double) | 卫星高度角掩蔽角 (°) | 15 |
| setDynamicModel(DynamicModel) | 动态模型：STATIC/CONST_SPEED/CONST_ACCELERATION | CONST_SPEED |
| setIonoOpt(IonoOpt) | 电离层选项：OFF/EST/IFLC | EST |
| setTropOpt(TropOpt) | 对流层选项：OFF/EST | EST |
| setWeights(WeightingStrategy) | 加权策略：EQUAL/SAT_ELEVATION/SNR/COMBINED | SAT_ELEVATION |
| setAmbiguityRatioThreshold(double) | LAMBDA Ratio 阈值 | 3.0 |
| setCycleSlipThreshold(double) | 周跳检测阈值 | 1.0 |
| setFixedHeight(Double) | 固定高程约束 (m) | null |
| setDebug(boolean) | 调试输出 | true |

**RunMode 枚举：**

| 模式 | 说明 |
|---|---|
| CODE_STANDALONE | 伪距单点定位（最小二乘） |
| CODE_DOUBLE_DIFF | 伪距双差定位（最小二乘） |
| KALMAN_FILTER_CODE_PHASE_STANDALONE | 伪距+相位单点卡尔曼滤波 |
| KALMAN_FILTER_CODE_PHASE_DOUBLE_DIFF | 伪距+相位双差卡尔曼滤波（RTK） |
| CODE_STANDALONE_SNAPSHOT | 单点快照定位 |
| CODE_STANDALONE_COARSETIME | 粗时单点定位 |

## 3. GNSS 位移计算接口

### 3.1 接口定义

**核心接口**：[DisplacementCalculator.java](file:///D:/code/goGPS_Java/src/main/java/org/gogpsproject/displacement/DisplacementCalculator.java)

```java
public interface DisplacementCalculator {
    // 单文件解算（单点定位）
    List<DisplacementResult> computeFromFile(String rtcmFilePath, double[] initPos, GnssConfig config);

    // 字节数组解算
    DisplacementResult computeFromBytes(byte[] rtcmData, double[] initPos, GnssConfig config);

    // 流式解算
    void computeFromStream(InputStream stream, double[] initPos, GnssConfig config, DisplacementCallback callback);

    // 双文件 RTK 解算（基站+流动站）
    List<DisplacementResult> computeFromFileWithBase(String roverFile, String baseFile,
                                                      double[] initPos, GnssConfig config);

    // 流式 RTK 解算
    void computeFromStreamWithBase(InputStream roverStream, InputStream baseStream,
                                    double[] initPos, GnssConfig config, DisplacementCallback callback);

    // 清洗：单条结果
    CleanResult cleanSingle(DisplacementResult result);

    // 清洗：带历史上下文
    CleanResult cleanWithHistory(DisplacementResult result, String deviceId);

    // 清洗：带气象数据
    CleanResult cleanWithHistoryAndMeta(DisplacementResult result, String deviceId, GnssMetaData meta);
}
```

### 3.2 离线 RTK 解算示例

```java
import org.gogpsproject.displacement.DefaultDisplacementCalculator;
import org.gogpsproject.displacement.config.*;
import org.gogpsproject.displacement.model.DisplacementResult;
import java.util.List;

// 1. 配置 GNSS 解算参数
GnssConfig gnssConfig = GnssConfig.builder()
    .elevationMask(15.0)                    // 高度角掩蔽角
    .tropoModel(TropoModel.EST)             // 实时估计对流层延迟
    .ionoModel(IonoModel.IFLC)              // 双频消电离层组合
    .onlyFixedSolution(true)                // 仅接受 FIX 解
    .minRatio(3.0)                          // LAMBDA Ratio 阈值
    .build();

// 2. 配置清洗参数
CleanConfig cleanConfig = new CleanConfig();
cleanConfig.maxStepHorizontal = 0.05;       // 水平跳变阈值 5cm
cleanConfig.maxStepVertical = 0.08;         // 高程跳变阈值 8cm
cleanConfig.windowSize = 20;                // 滑动窗口大小
cleanConfig.algorithm = Algorithm.IQR;       // 粗差检测算法

// 3. 配置诊断参数
DiagnosisConfig diagnosisConfig = new DiagnosisConfig();

// 4. 配置缓存参数
CacheConfig cacheConfig = new CacheConfig();
cacheConfig.maxDevices = 100;               // 最大设备数

// 5. 创建解算器
DefaultDisplacementCalculator calculator = new DefaultDisplacementCalculator(
    gnssConfig, cleanConfig, diagnosisConfig, cacheConfig);

// 6. 执行离线 RTK 解算
double[] initPos = {0, 0, 0};              // 基站初始位置 (ECEF)
List<DisplacementResult> results = calculator.computeFromFileWithBase(
    "rover.rtcm3", "base.rtcm3", initPos, gnssConfig);

// 7. 遍历结果
for (DisplacementResult r : results) {
    System.out.printf("NEU: [%.4f, %.4f, %.4f] status=%s sat=%d PDOP=%.2f%n",
        r.getdNorth(), r.getdEast(), r.getdUp(),
        r.getStatus(), r.getNumSatellites(), r.getPdop());
}
```

### 3.4 带持久化回调的完整示例

持久化是**清洗流水线**的内置能力，通过 [PersistenceCallback](file:///D:/code/goGPS_Java/src/main/java/org/gogpsproject/displacement/persistence/PersistenceCallback.java) 接口解耦存储实现。清洗过程中自动触发：结果保存、设备状态快照、异常事件记录、诊断报告存储。

**接口定义**（共 13 个方法）：

```
PersistenceCallback
├── 清洗结果持久化
│   ├── saveResult(deviceId, DisplacementResult)
│   ├── queryRecentResults(deviceId, count)
│   └── queryTimeRange(deviceId, start, end)
├── 设备状态持久化
│   ├── saveDeviceState(deviceId, DeviceStateSnapshot)
│   ├── loadDeviceState(deviceId)
│   └── hasHistory(deviceId)
├── 异常事件持久化
│   ├── saveAnomalyEvent(deviceId, AnomalyEvent)
│   └── queryRecentAnomalies(deviceId, count)
├── 诊断报告持久化
│   ├── saveDiagnosis(deviceId, DeviceDiagnosis)
│   └── queryLatestDiagnosis(deviceId)
├── 修正记录持久化
│   ├── saveCorrectionRecord(deviceId, DataCorrectionRecord)
│   ├── queryCorrectionRecords(deviceId)
│   └── queryCorrectedData(deviceId, dataId)
└── 删除
    └── deleteDeviceHistory(deviceId)
```

**完整示例**：

```java
import org.gogpsproject.displacement.DefaultDisplacementCalculator;
import org.gogpsproject.displacement.config.*;
import org.gogpsproject.displacement.model.*;
import org.gogpsproject.displacement.persistence.PersistenceCallback;
import java.time.Instant;
import java.util.List;

// ========== 1. 实现持久化回调接口（对接任意存储层） ==========
PersistenceCallback callback = new PersistenceCallback() {

    // ----- 清洗结果 -----
    @Override
    public void saveResult(String deviceId, DisplacementResult result) {
        // 每条清洗后的结果都回调此方法，可写入 MySQL/Redis/InfluxDB
        // result 包含原始值、清洗值、dataId、异常标记等完整信息
        System.out.printf("[持久化] 设备=%s dataId=%s NEU=[%.4f,%.4f,%.4f] 异常=%s%n",
            deviceId, result.getDataId(),
            result.getdNorth(), result.getdEast(), result.getdUp(),
            result.isAbnormal());
    }

    @Override
    public List<DisplacementResult> queryRecentResults(String deviceId, int count) {
        // 查询最近N条结果，用于恢复滑动窗口
        return List.of(); // 示例返回空，实际从数据库查询
    }

    @Override
    public List<DisplacementResult> queryTimeRange(String deviceId, Instant start, Instant end) {
        return List.of();
    }

    // ----- 设备状态 -----
    @Override
    public void saveDeviceState(String deviceId, DeviceStateSnapshot snapshot) {
        // LRU 淘汰时自动调用，保存滑动窗口、双基线、温度状态等
        System.out.printf("[持久化] 设备=%s 状态快照已保存 (基线N=%.4f)%n",
            deviceId, snapshot.getFastBaselineNorth());
    }

    @Override
    public DeviceStateSnapshot loadDeviceState(String deviceId) {
        // 设备重新活跃时调用，恢复上次淘汰的内存状态
        return null; // 示例返回null，实际从数据库恢复
    }

    @Override
    public boolean hasHistory(String deviceId) {
        return false;
    }

    // ----- 异常事件 -----
    @Override
    public void saveAnomalyEvent(String deviceId, AnomalyEvent event) {
        // 清洗过程检测到异常时自动回调
        System.out.printf("[持久化] 设备=%s 异常事件: %s%n", deviceId, event.getDescription());
    }

    @Override
    public List<AnomalyEvent> queryRecentAnomalies(String deviceId, int count) {
        return List.of();
    }

    // ----- 诊断报告 -----
    @Override
    public void saveDiagnosis(String deviceId, DeviceDiagnosis diagnosis) {
        // 诊断模块生成报告后回调
    }

    @Override
    public DeviceDiagnosis queryLatestDiagnosis(String deviceId) {
        return null;
    }

    // ----- 修正记录 -----
    @Override
    public void saveCorrectionRecord(String deviceId, DataCorrectionRecord record) {
        // 诊断发现前期误判，生成修正记录时回调
    }

    @Override
    public List<DataCorrectionRecord> queryCorrectionRecords(String deviceId) {
        return List.of();
    }

    @Override
    public DisplacementResult queryCorrectedData(String deviceId, String dataId) {
        return null;
    }

    // ----- 删除 -----
    @Override
    public void deleteDeviceHistory(String deviceId) {
        // 清理设备所有历史数据
    }
};

// ========== 2. 启用持久化 ==========
PersistenceConfig persistenceConfig = new PersistenceConfig();
persistenceConfig.enablePersistence = true;

// ========== 3. 创建带持久化支持的解算器 ==========
DefaultDisplacementCalculator calculator = new DefaultDisplacementCalculator(
    gnssConfig, cleanConfig, diagnosisConfig, cacheConfig,
    persistenceConfig, callback);

// ========== 4. 执行解算+清洗（持久化自动触发） ==========
String deviceId = "DEVICE_001";
for (DisplacementResult result : results) {
    // 清洗过程中自动调用 saveResult / saveAnomalyEvent / saveDeviceState 等
    CleanResult cleanResult = calculator.cleanWithHistory(result, deviceId);
    System.out.printf("设备=%s 清洗结果=%s%n", deviceId, cleanResult.isPassed());
}

// ========== 5. 清除设备状态时自动持久化快照 ==========
calculator.clearDeviceState(deviceId);  // 内部调用 saveDeviceState
```

**持久化触发时机**：

| 触发场景 | 回调方法 | 说明 |
|---|---|---|
| 每次清洗完成 | saveResult | 保存清洗后的位移结果 |
| 检测到异常 | saveAnomalyEvent | 保存异常事件记录 |
| LRU 淘汰设备 | saveDeviceState | 淘汰前保存状态快照 |
| 主动清除设备 | saveDeviceState | 清除前保存状态快照 |
| 诊断完成 | saveDiagnosis | 保存诊断报告 |
| 发现前期误判 | saveCorrectionRecord | 保存修正记录 |

### 3.5 DisplacementResult 关键字段

| 字段 | 类型 | 说明 |
|---|---|---|
| dataId | String | 数据唯一 ID（UTC 毫秒+序列号） |
| timestamp | Instant | 数据时间戳 |
| dNorth / dEast / dUp | double | NEU 位移量 (m) |
| status | SolutionStatus | 解算状态：FIX / FLOAT / INVALID |
| numSatellites | int | 参与解算卫星数 |
| pdop | double | 位置精度因子 |
| rms | double | 均方根误差 |
| ratio | double | LAMBDA Ratio 值 |
| abnormal | boolean | 是否异常 |
| abnormalReason | String | 异常原因 |

## 4. 清洗流水线接口

### 4.1 单条清洗（无历史）

```java
// 适用于无历史数据的首条结果
CleanResult cleanResult = calculator.cleanSingle(result);

if (cleanResult.isPassed()) {
    System.out.println("清洗通过");
} else {
    System.out.println("清洗失败: " + cleanResult.getReason());
    System.out.println("失败层级: " + cleanResult.getLayer());
}
```

### 4.2 带历史上下文的五层清洗

```java
String deviceId = "DEVICE_001";

for (DisplacementResult result : results) {
    CleanResult cleanResult = calculator.cleanWithHistory(result, deviceId);

    if (cleanResult.isPassed()) {
        // 通过：结果可信
    } else {
        // 被拒绝：记录异常
        System.out.printf("历元 %s 清洗失败: %s%n",
            result.getTimestamp(), cleanResult.getReason());
    }
}
```
