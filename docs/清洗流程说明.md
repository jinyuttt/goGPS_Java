# 清洗流程功能说明

## 概述

清洗流水线是位移解算引擎的核心质量控制模块，对 GNSS RTK 解算得到的 NEU 位移结果进行六层递进过滤。前五层为单测点时序质控，逐层排除异常值；第六层为空间一致性批量后置质控，独立于前五层流程。确保输出数据的可靠性。

**核心类**：[DisplacementCleaner.java](file:///D:/goGPS_Java/src/main/java/org/gogpsproject/displacement/cleaning/DisplacementCleaner.java)

## 整体架构

```
GNSS RTK 解算结果 (NEU)
        │
        ▼
┌───────────────────┐
│  第1层：质量门禁    │  ← 卫星数 / PDOP / RMS / Ratio
├───────────────────┤
│  第2层：跳变检测    │  ← 相邻历元差分 + 同向趋势快速通道
├───────────────────┤
│  第3层：统计粗差    │  ← 滑动窗口 IQR / 3σ
├───────────────────┤
│  第4层：值替换      │  ← 中位数/均值替换（仅第3层触发，置 isTimeSeriesCorrected）
├───────────────────┤
│  第5层：基线记忆    │  ← 快慢基线对比 + 阶跃观察（系统偏差校准，不标记时序修正）
└───────────────────┘
        │ 前五层单测点时序质控完成
        ▼
   清洗结果 (CleanResult)
        │
        ▼
┌───────────────────┐
│  第6层：空间一致性  │  ← 同历元同组多测点批量后置质控
│  校验（批量后置）   │  ← IDW插值离群判定 + 群体一致性 + 雪崩防护
└───────────────────┘
        │
        ▼
┌───────────────────┐
│   设备诊断模块      │  ← 温漂/周期/抖动分析
│   (DeviceDiagnostician)  │
└───────────────────┘
        │
        ▼
   自适应阈值反馈
    (Adjust Thresholds)
```

> **说明**：第6层为独立批量后置接口，需通过 `layer6SpatialConsistencyBatch()` 单独调用，不在 `cleanWithHistory()` 单测点流程内执行。

## 逐层详解

### 第一层：质量门禁 (Layer 1 — Quality Gate)

**目的**：过滤 GNSS 解算质量不合格的结果。

**检测规则**：

| 解算状态 | 检查项 | 阈值（默认） | 不通过行为 |
|---|---|---|---|
| INVALID/SINGLE | — | — | 直接拒绝 |
| FLOAT | 卫星数 | < 4 | 标记异常，拒绝 |
| FLOAT | PDOP | > 10.0 | 标记异常，拒绝 |
| FLOAT | RMS | > 0.05 | 标记异常，拒绝 |
| FIX | 卫星数 | < 4 | 标记异常，拒绝 |
| FIX | PDOP | > 6.0 | 标记异常，拒绝 |
| FIX | RMS | > 0.03 | 标记异常，拒绝 |
| FIX | Ratio | < 3.0 | 标记异常，拒绝 |

**实现位置**：[layer1QualityGate()](file:///D:/goGPS_Java/src/main/java/org/gogpsproject/displacement/cleaning/DisplacementCleaner.java#L222-L279)

### 第二层：跳变检测 (Layer 2 — Jump Detection)

**目的**：检测相邻历元间的位移突变，识别异常跳变。

**算法**：

1. 计算当前值与上一合法值的差分 ΔN、ΔE、ΔU
2. 若任一方向超出跳变阈值：
   - 水平阈值：默认 0.05m（5cm）
   - 高程阈值：默认 0.08m（8cm）
3. 触发跳变标记

**同向趋势快速通道**：

```
跳变检测触发
    │
    ▼
计算跳变方向符号 (signN, signE, signU)
    │
    ▼
检查历史跳变方向是否一致
    │
    ├── 连续 N 个历元同向跳变 → 判定为真实形变 → 放行
    │
    └── 方向不一致 → 标记为跳变异常
```

该机制避免了真实形变（如滑坡、建筑沉降）被误判为异常跳变。

**关键配置**：

| 参数 | 默认值 | 说明 |
|---|---|---|
| maxStepHorizontal | 0.05 m | 水平跳变阈值 |
| maxStepVertical | 0.08 m | 高程跳变阈值 |
| trendQuickReleaseEnabled | true | 是否启用快速通道 |
| trendQuickReleaseCount | 5 | 需要连续同向历元数 |
| trendQuickReleaseMaxSeconds | 300 | 快速通道时间窗口（秒） |

**实现位置**：[layer2JumpDetection()](file:///D:/goGPS_Java/src/main/java/org/gogpsproject/displacement/cleaning/DisplacementCleaner.java#L303-L391)

### 第三层：统计粗差 (Layer 3 — Statistical Outlier)

**目的**：基于滑动窗口统计模型检测孤立粗差。

**算法选择**：

**IQR 法**（默认）：
```
Q1 = 窗口第25百分位数
Q3 = 窗口第75百分位数
IQR = Q3 - Q1
粗差判断：值 ∉ [Q1 - 1.5×IQR, Q3 + 1.5×IQR]
```

**3σ 法**：
```
μ = 窗口均值
σ = 窗口标准差
粗差判断：|值 - μ| > 3σ
```

**高程方向**：IQR 系数乘以 `verticalToleranceFactor`（默认 1.5），允许更大的高程波动。

**窗口未满时的保守策略**：

- 初始基线已建立 → 与初始基线比较，超 3σ 视为粗差
- 窗口 < 3 条 → 直接放行

**关键配置**：

| 参数 | 默认值 | 说明 |
|---|---|---|
| windowSize | 20 | 滑动窗口大小 |
| algorithm | IQR | 统计算法：IQR / 3σ |
| verticalToleranceFactor | 1.5 | 高程容忍系数 |
| windowInitFallbackSigma | 0.01 | 窗口未满时回退 σ 值 |

**实现位置**：[layer3StatisticalOutlier()](file:///D:/goGPS_Java/src/main/java/org/gogpsproject/displacement/cleaning/DisplacementCleaner.java#L407-L456)

### 第四层：值替换 (Layer 4 — Anomaly Replacement)

**目的**：将第3层检测到的粗差值替换为合理的统计估计值，使其可继续参与后续计算。

**替换策略**：

| 窗口状态 | IQR 模式 | 3σ 模式 |
|---|---|---|
| 窗口已满（≥ windowSize） | 替换为窗口中位数 | 替换为窗口均值 |
| 窗口未满 | 替换为上一合法值 | 替换为上一合法值 |

**时序修正标记**（增量修改）：

值替换实际执行时，会设置以下标记，实现修正行为可追溯：
- `CleanResult.isTimeSeriesCorrected` 置为 `true`
- `statusFlags` 追加「时序粗差替换」标识

**注意**：第4层仅在**第3层检测到粗差**时触发，不会独立运行。替换后的值会清除异常标记，使其能正常参与第五层和窗口更新。第5层阶跃回正属于系统偏差校准，不属于异常替换，不标记时序修正。

**实现位置**：[layer4AnomalyReplacement()](file:///D:/goGPS_Java/src/main/java/org/gogpsproject/displacement/cleaning/DisplacementCleaner.java#L547-L599)

### 第五层：基线记忆 (Layer 5 — Baseline Memory)

**目的**：检测长期阶跃偏移，区分真实位移与系统偏差。

**双基线机制**：

| 基线 | 窗口大小 | 用途 |
|---|---|---|
| 快速基线 (fastBaseline) | 约 300 历元（5 分钟 @ 1Hz） | 反映短期变化 |
| 慢速基线 (slowBaseline) | 约 86400 历元（24 小时 @ 1Hz） | 反映长期趋势 |

**阶跃检测三阶段流程**：

```
阶段1：检测大幅跳变
    │  差分 > stepDeviationThreshold (默认 3cm)
    │
    ▼
阶段2：观察期（累计稳定历元）
    │  连续 stepObservationCount 个历元稳定在候选值附近
    │  → 超时保护：stepMaxTimeoutSeconds 内未确认则放弃
    │
    ▼
阶段3：确认与回正
    │  快慢基线中位数差异 < stepConfirmThreshold (默认 2mm)
    │  → 确认为系统阶跃，回正到窗口中位数
    │  → 差异过大则放弃
```

**关键配置**：

| 参数 | 默认值 | 说明 |
|---|---|---|
| stepDeviationThreshold | 0.03 m | 阶跃检测阈值 |
| stepObservationCount | 10 | 观察期需稳定历元数 |
| stepStableThreshold | 0.005 m | 稳定判定阈值 |
| stepConfirmThreshold | 0.002 m | 快慢基线确认阈值 |
| stepMaxTimeoutSeconds | 300 | 观察期超时（秒） |
| longTermWindowSize | 300 | 快速基线窗口大小 |
| slowBaselineSize | 86400 | 慢速基线窗口大小 |

**实现位置**：[layer5BaselineMemory()](file:///D:/goGPS_Java/src/main/java/org/gogpsproject/displacement/cleaning/DisplacementCleaner.java#L613-L672)

### 第六层：空间一致性校验 (Layer 6 — Spatial Consistency)

**目的**：对同历元、同监测组的所有测点进行批量后置质控，识别空间孤立噪声、群体真实形变与基准站漂移，提升区域级数据可靠性。

**接口定义**：

```java
public List<CleanResult> layer6SpatialConsistencyBatch(
    String groupId, long epochTime, List<CleanResult> preCleanedResults)
```

> **调用约束**：输入必须为同一历元、同一监测组的所有测点前五层输出；输出与输入顺序严格对应；`dataId`、`stationId`、`time`、原始解算状态全程透传。

**基准坐标初始化**：

| 取值优先级 | 规则 |
|---|---|
| 人工配置坐标 | 通过 `setStationBaseCoord()` 设置，优先级最高，自动逻辑永不覆盖 |
| 系统自动生成 | 连续 30 个 FIX 固定解历元定位结果取均值，作为永久基准坐标缓存 |
| 初始化失败 | 测点自身跳过空间校验，也不参与其他测点的邻站计算 |

**邻站权重计算**：

```
综合权重 = 静态空间权重（IDW预计算缓存） × 动态质量权重（实时匹配）
```

- **静态空间权重**：基于基准坐标计算两两平面距离，超出 `spatialValidDistance` 置 0，按 IDW 公式 `w = 1/d^power` 预计算并按组缓存
- **动态质量权重**：FIX=1.0 / FLOAT=0.2 / SINGLE/INVALID/前五层标记无效=0
- **归一化**：对有效权重做归一化，总权重和为 1

**空间离群判定与修正**：

```
1. IDW插值计算理论预期位移（邻站加权求和）
2. 计算实际位移与预期值的残差
3. 水平残差 > spatialOutlierHorizontal 或 垂向残差 > spatialOutlierVertical → 判定离群
```

| 测点类型 | 连续修正次数 | 处理方式 | 标记 |
|---|---|---|---|
| NORMAL | < 上限 | 替换为 `spatialLastValid*` 历史正常值 | `isSpatialCorrected=true` + 空间离群修正 |
| NORMAL | ≥ 上限 | 原样输出，停止修正 | 连续修正超限 |
| SPECIAL | — | 原样输出，特赦放行 | 裂缝独立形变 |
| 正常通过 | — | 更新 `spatialLastValid*`，清零计数器 | 空间校验正常 |

**群体一致性判定**（仅打标识，不改数值）：

| 判定类型 | 条件 | 标识 |
|---|---|---|
| 局部真实形变 | 同向邻站加权占比 ≥ `spatialDeformationRatio` | 局部真实形变 |
| 基准站漂移疑似 | 整组三向位移标准差均 < `spatialDriftDiffThreshold` | 基准站漂移疑似 |
| 平票兜底 | 同向与反向加权占比差值 < 10% | 平票兜底 |

**兜底规则**：

| 兜底类型 | 触发条件 | 处理方式 | 标识 |
|---|---|---|---|
| 区域质量劣化 | 组内非 FIX 解占比 ≥ `spatialBadQualityRatio` | 所有测点跳过修正 | 区域劣化跳过校验 |
| 邻站不足 | 有效邻站数 < `spatialMinNeighborCount` 或权重和为 0 | 跳过空间校验 | 邻站不足跳过校验 |

**关键配置**：

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| spatialValidDistance | double | 300.0 | 邻站有效判定距离 (m) |
| spatialIdwPower | int | 2 | IDW 距离衰减幂指数 |
| spatialOutlierHorizontal | double | 0.015 | 水平残差阈值 (m) |
| spatialOutlierVertical | double | 0.030 | 垂向残差阈值 (m) |
| spatialMaxCorrectionCount | int | 5 | 单测点连续空间修正最大次数 |
| spatialMinNeighborCount | int | 3 | 最小有效邻站数量 |
| spatialDeformationRatio | double | 0.7 | 局部真实形变同向占比阈值 |
| spatialDriftDiffThreshold | double | 0.005 | 基准站漂移标准差阈值 (m) |
| spatialBadQualityRatio | double | 0.5 | 区域劣化低质量占比阈值 |
| spatialDirectionTolerance | double | 30.0 | 同向位移方向容忍角度 (°) |

**配套管理接口**：

| 方法 | 用途 |
|---|---|
| `registerStationToGroup(groupId, stationId)` | 注册测点到监测组 |
| `setStationBaseCoord(stationId, x, y)` | 人工配置基准坐标 |
| `setStationType(stationId, type)` | 设置测点类型（NORMAL/SPECIAL） |
| `refreshSpatialBaseCoords(groupId)` | 全量刷新组基准坐标与权重缓存 |
| `refreshSingleSpatialBaseCoord(stationId)` | 刷新单个测点基准坐标与权重 |

**实现位置**：[layer6SpatialConsistencyBatch()](file:///D:/goGPS_Java/src/main/java/org/gogpsproject/displacement/cleaning/DisplacementCleaner.java#L804-L920)

## 自适应阈值

### 诊断反馈机制

清洗器支持接收设备诊断模块的反馈，动态调整阈值：

```
设备诊断 (DeviceDiagnostician)
    │
    ├── 温漂模式：温度-位移相关性显著
    │     → 水平跳变阈值 × 1.6
    │     → 高程跳变阈值 × 1.5
    │     → IQR 系数 × 2.0
    │
    ├── 周期波动模式：当前处于高波动时段
    │     → 水平跳变阈值 × 1.6
    │     → 高程跳变阈值 × 1.5
    │     → IQR 系数 × 2.0
    │
    └── 抖动模式：仅标记，不调整阈值
```

**实现位置**：[applyDiagnosisFeedback()](file:///D:/goGPS_Java/src/main/java/org/gogpsproject/displacement/cleaning/DisplacementCleaner.java#L78-L97)

**关键配置**：

| 参数 | 默认值 | 说明 |
|---|---|---|
| adaptiveThresholdEnabled | false | 是否启用自适应阈值 |
| thermalCorrelationThreshold | 0.6 | 温漂相关性阈值 |
| thermalModeScaleHorizontal | 1.6 | 温漂模式水平缩放 |
| thermalModeScaleVertical | 1.5 | 温漂模式高程缩放 |
| thermalIqrScaleMultiplier | 2.0 | 温漂模式 IQR 缩放 |

## 数据流示例

以一次完整历元处理为例：

```
输入: DisplacementResult {
    dNorth=0.0012, dEast=-0.0008, dUp=0.0025
    status=FIX, numSatellites=12, pdop=1.8, rms=0.005, ratio=5.2
}

第1层 质量门禁:
    FIX解, sat=12≥4✓, PDOP=1.8≤6.0✓, RMS=0.005≤0.03✓, Ratio=5.2≥3.0✓
    → 通过

第2层 跳变检测:
    上一合法值: (0.0010, -0.0007, 0.0023)
    差分: ΔN=0.0002, ΔE=0.0001, ΔU=0.0002
    全部 < 阈值(0.05, 0.05, 0.08)
    → 通过，重置同向趋势

第3层 统计粗差:
    窗口 (20条): [0.0010, 0.0011, 0.0009, ...]
    IQR: Q1=0.0009, Q3=0.0013, IQR=0.0004
    区间: [-0.0004, 0.0026]
    dNorth=0.0012 ∈ 区间 ✓
    → 通过

第4层 值替换:
    未触发（第3层通过）
    → 跳过

第5层 基线记忆:
    无大幅跳变
    → 通过

前五层单测点时序质控完成: CleanResult.pass()
```

### 第6层批量后置调用示例

前五层完成后，由调度层按历元聚合同组所有测点结果，统一调用第6层：

```
输入: 同一历元、同一监测组所有测点的前五层 CleanResult 列表
      groupId="groupA", epochTime=1716969600000
      [
        {stationId="S01", dNorth=0.0012, dEast=-0.0008, dUp=0.0025, status=FIX},
        {stationId="S02", dNorth=0.0011, dEast=-0.0007, dUp=0.0023, status=FIX},
        {stationId="S03", dNorth=0.0050, dEast=-0.0005, dUp=0.0020, status=FIX},  ← 疑似离群
        {stationId="S04", dNorth=0.0013, dEast=-0.0009, dUp=0.0024, status=FIX}
      ]

第6层 空间一致性校验:
    1. 基准坐标检查: S01~S04 均已初始化
    2. 邻站权重: IDW 预计算缓存命中，按距离归一化
    3. 动态质量权重: 全部 FIX → 权重=1.0
    4. IDW 插值预期位移:
       S03 预期 = Σ(邻站位移 × 权重) ≈ (0.0012, -0.0008, 0.0024)
       S03 实际 = (0.0050, -0.0005, 0.0020)
       水平残差 = 0.0038 > spatialOutlierHorizontal(0.015) ❌
       → 判定离群，替换为 spatialLastValid*
       → isSpatialCorrected=true，连续修正计数 +1
    5. 其他测点残差均 < 阈值 → 通过，更新 spatialLastValid*

最终: 输出 4 条 CleanResult，S03 被空间修正，其余原样透传
```

## 设备状态管理

清洗过程中维护的设备状态（[DeviceState](file:///D:/goGPS_Java/src/main/java/org/gogpsproject/displacement/model/DeviceState.java)）：

| 状态字段 | 用途 |
|---|---|
| lastValidNorth/East/Up | 上一合法值，用于跳变检测 |
| northWindow/eastWindow/upWindow | 滑动窗口，用于统计粗差 |
| fastBaseline | 快速基线（5分钟），用于阶跃确认 |
| slowBaseline | 慢速基线（24小时），用于长期趋势 |
| lastTemperature | 当前温度，用于温漂分析 |
| initialBaseline* | 初始基线，用于窗口未满时回退 |
| stepObservation* | 阶跃观察状态 |
| jumpDirectionHistory | 跳变方向历史，用于同向趋势判断 |
| lastDiagnosis | 上次诊断结果，用于自适应阈值 |

> **第6层独立状态**：空间一致性校验的状态（`spatialLastValid*`、`spatialCorrectionCount`、基准坐标缓存、IDW 权重缓存）由 `DisplacementCleaner` 内部维护，**不写入** `DeviceState`，避免污染前五层单测点时序状态。第6层仅修改输出 `CleanResult`，绝不回写 `DeviceState` 中前五层状态字段。

状态通过 [DeviceStateCache](file:///D:/goGPS_Java/src/main/java/org/gogpsproject/displacement/cache/DeviceStateCache.java) 以 LRU 策略管理，淘汰时自动持久化到 [DeviceStateSnapshot](file:///D:/goGPS_Java/src/main/java/org/gogpsproject/displacement/model/DeviceStateSnapshot.java)。
